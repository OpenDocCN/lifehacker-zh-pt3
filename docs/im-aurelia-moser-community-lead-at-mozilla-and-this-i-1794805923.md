# 我是 Aurelia Moser，Mozilla 的社区负责人，这是我的工作方式

> 原文：<https://lifehacker.com/im-aurelia-moser-community-lead-at-mozilla-and-this-i-1794805923>

Aurelia Moser 是一名开发人员、教师、作者，同时也是一名制图师。她在 [Mozilla 科学实验室](https://science.mozilla.org/) 与研究人员合作开源项目，同时也是 [女孩开发 It](https://www.girldevelopit.com/chapters/new-york) 的导师。



Mozilla 科学实验室是一个由程序员和科学家组成的社区，他们一起工作，分享他们的数据，并本着与 Mozilla 相同的开源精神进行合作。除了她在 [SVA-DSI](http://dsi.sva.edu/) 和 [帕森斯](http://www.newschool.edu/lang/journalism-design/) 的教学之外，奥雷利亚是那些似乎同时有十份工作，但不知何故通过颜色编码、文本编辑器和 IRC 系统保持组织的人之一。她是这样工作的。

**一个词最能描述你的工作方式:**连续地，以图表方式？
**当前移动设备:**屏幕开裂的 iPhone 6
**当前电脑:**带摄像头的 MacBook Air (2015)

#### 首先，简单介绍一下你的背景，以及你是如何走到今天的。你是怎么加入 Mozilla 的？

我的背景相当丰富多彩，学术上和职业上。我去了艺术和媒体保护的研究生院，包括复兴原生数字和新媒体艺术项目，供图书馆和博物馆的顾客持续展示和观看。这是对技术取证的一个非常好的介绍，诊断代码问题并向后工作，为损坏的应用程序、废弃的数据存储格式和废弃的 API 开发仿真器和解决方案。

我在一些创新技术机构工作，然后申请成为 Mozilla 和 [Ushahidi](https://www.ushahidi.com/) 的公开新闻研究员，这是一家东非技术公司，致力于开发由功能和智能手机数据驱动的全球规模的危机地图软件。从那以后，我对地图产生了浓厚的兴趣，并为 [Carto](https://carto.com/) 和一些地理空间软件初创公司工作，之后又回到了科学领域，并申请在 [Mozilla 科学实验室](https://science.mozilla.org/) 工作，这是 Mozilla 基金会和更广泛的网络的一部分，旨在支持希望开源他们的研究(开放数据、开放访问出版、开放科学实践)的科学家。这让我感到有趣和满足，而且不完全是我在科学保护方面的第三阶段教育；开放科学和开放源代码为促进和保存研究成果提供了一些途径...给科学家一些分享想法和更快地重复想法的代理，从网络上更广泛的研究社区的反馈中获得信息。

#### 你离不开哪些应用、软件或工具？

*   ——文字编辑器的选择，有一个自定义的颜色主题，我每隔几个月就更换一次。

*   [【Mou】](http://25.io/mou/)——并排降价预览应用，在这里我记笔记和存储文本文件。

*   [Firefox](https://testpilot.firefox.com/)//[Chrome](https://www.google.com/chrome/browser/desktop/index.html?brand=CHBD&gclid=EAIaIQobChMIoYvntNHK0wIVmY2zCh2yagk2EAAYASAAEgI6r_D_BwE)/[Safari](https://www.apple.com/safari/)——所有用于测试的浏览器，有时用于在同一平台上同时运行多个账号。

*   [座谈会](http://colloquy.info/)——IRC 的应用，我们仍然在 Mozilla 内部使用，尽管 Slack/Gitter/MatterMost/Adium 最近相当流行。

*   喜欢这个设置键盘热键的应用程序，它可以在你的屏幕上切换窗口和应用程序，对任何一个小屏幕和严重节省空间的人都很有用。

*   对我来说，动画 gif 是一种完整的计算语言，我用它们来说明错误，记录拉取请求功能，展示编程概念，并捕捉令人愉快的日常交互。这个应用程序可以帮助你在任何屏幕上做到这一点；无法解释这款应用奇怪的名字，但它的实用性是一流的。

*   [邮件](https://support.apple.com/mail)——我不喜欢很多本地邮件客户端，但我使用苹果的邮件(和以前的雷鸟)进行 PGP 加密通信。

*   iOS 上的闹铃——大概是我真的离不开的东西:我的手机闹铃。我设置了多个闹钟，用很多音调来吓我刺激我。

*   ——一个可爱的安排邮件通信的应用程序。我是收件箱零英雄，但我不喜欢把我奇怪的工作时间强加给别人。

*   我还尽可能在每个应用程序上安装一个“黑暗主题”，让我的眼睛稍微放松一下。我真的不喜欢 [f.lux](https://justgetflux.com/) ，或者任何在我的屏幕上联合着色的东西，因为它扭曲了许多颜色和设计考虑，但我确实认为从背光中获得视觉休息是好的。

#### 你的工作空间是什么样的？有笔记本电脑和耳机的咖啡店？带立式办公桌的家庭办公室？

在 Mozilla，我们有远程工作的特权，所以我经常在家工作。我的 twee 公寓在客厅里有一张小桌子，这是我典型的工作环境。Mozilla 在布鲁克林的 DUMBO 也有一个小小的共同工作空间，如果我想要稍微快一点的 Wi-Fi，我可以骑自行车去那里。耳机全天/一直；我每月只吃一副耳机，这可能是对大多数耳机硬件质量差的可悲控诉，而不是我过度使用耳塞。

#### 去生活帮最省时的捷径是什么？

我试过 [番茄](https://en.wikipedia.org/wiki/Pomodoro_Technique) 计划，但有时发现它给我的一天增加了不必要的压力。作为一种改进，我通常尝试立即做任何不到两分钟的事情，我用可爱的方式与自己竞争，试图在两分钟的冲刺中最大化我的生产力。我还会打开多个浏览器，并在它们之间按主题分配任务，所以如果我需要专注于一项任务，我会关闭浏览器并加载我的电子邮件客户端，以避免分心和电子邮件通知。

#### 你最喜欢的待办事项管理器是什么？

我真的很喜欢，主要是因为我可以定制和颜色编码它，而且我很喜欢颜色编码。虽然它在你的终端上，所以最适合私人待办事项列表，而且条目是手动的，所以维护起来很有挑战性。

[GitHub 项目板](https://help.github.com/articles/about-project-boards/) 牛逼；我对在 GitHub 中组织我的问题和任务有些神经质，所以我经常将我的待办事项列表转移到 GitHub repos，并为我的每一项工作建立一个新的 repo 和任务日志。

#### 你在日常生活中的哪些方面比其他人做得更好？你的秘诀是什么？

购买奇怪的域名，在不适当的时候笑，同时吹口哨和哼歌，吸鼻子(在不适当的时候笑的必然结果)。

但说真的，我过度组织，真的很擅长颜色编码。我把我的手机应用程序、衣柜里的衣服、书架归类并涂上颜色...至少对我来说， [ROYGBIV 发行版](https://www.pinterest.com/aure_moser/colorific/) 有一些非常令人愉快的地方。这听起来很可能是假的，但我有彩色记忆:我可以很容易地通过联想颜色、我公寓里东西的位置、它们的功能和用途来回忆事情...就像无用的联觉或记忆术。

#### 你工作的时候听什么？有最喜欢的播放列表吗？也许谈电台？还是更喜欢沉默？

我用对音乐进行分层，往往是 Spotify 上别人的播放列表。比起通过拇指按钮训练 Spotify 或 Pandora 得到的由算法生成的偏好播放列表，我更喜欢在世界各地找到让我感兴趣的音乐收藏的人这种奇怪的意外收获。如果我需要集中注意力，它必须是古典音乐或器乐；但除此之外，我通常喜欢 60/70/80 年代的 shoegaze、twee indie、法国和西班牙摇滚乐。

我喜欢[radio oooo](http://radiooooo.com/)的时空曲目 [Pandora](https://www.pandora.com/) 的随意播放列表生成。我在 Spotify 上只有几个播放列表；这可能是我现在最喜欢的。我对音乐有很强的情感依恋，所以如果我在做某件事的时候听一首歌，那件事会在音乐上留下印记，我很难随便听同一首歌。例如，我的跑步播放列表上的几乎任何东西都是“安全”的曲目，我不可能在听的时候不触发一种心脏震荡或轻度恐慌发作。

#### **怎么充值？当你想忘记工作的时候你会怎么做？**

当我想休息的时候，我会做瑜伽，或者有时去上芭蕾课，或者洗个澡。我是一个糟糕的冥想者，尽管我已经尝试了很多次。我的睡眠时间表可笑地不稳定，所以我不能说我以那种方式充电，尽管我是一个激烈和暴力的梦想家；我有过几次清醒梦的经历，让我感到莫名其妙地充满活力。

#### 你的睡眠规律是怎样的？你是夜猫子还是早起者？

我会称自己为“低级睡眠者”用更委婉的话来说，我可能有资格称之为“更进化”我每天晚上睡大约 5 个小时，尽管我真的想睡得更多。我几乎没有睡觉的习惯；有时它们包括 ASMR YouTube 视频、热茶、夜间瑜伽、褪黑素小熊软糖。我真的很喜欢在早上 8 点前起床，并且会优先于几乎所有的深夜活动。

#### 你收到过的最好的建议是什么？

“你花的时间不是你自己的。”

保罗·福特的 [10 个时间框架](http://contentsmagazine.com/articles/10-timeframes/) ，这是一个非常精彩的毕业演讲，他讨论了作为一个创造者或开发者，你有义务考虑你的应用和设计中隐含的时间消耗。你应该想一想，当别人使用你的作品时，你浪费了他们多少时间，因为这种浪费，如果结构不合理，对人类的进步是如此有害，对人类来说是如此可悲的债务。务实的程序员尊重他人的时间。

我对时间抽象和研究我们如何处理和理解时间的流逝非常感兴趣，时间是我们最令人沮丧的不可再生资源。我提议进行关于时区的会议讨论，却因为与时间相关的编程错误而浪费了很多时间。
