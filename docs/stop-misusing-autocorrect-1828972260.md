# 停止误用自动更正

> 原文:[https://lifehacker.com/stop-misusing-autocorrect-1828972260](https://lifehacker.com/stop-misusing-autocorrect-1828972260)

发明手机自动更正功能的人说，使用手机自动更正功能有正确的方法和错误的方法。肯·柯辛达(Ken Kocienda)，前苹果 iPhone 软件首席工程师，新书《创意精选:乔布斯 黄金时代的苹果设计过程内幕》的作者，带领团队为 iOS 首次发布创造了第一个软件键盘。他说自动更正可能会变得过于激进。

“自动更正”自其第一种形式出现以来已经有所发展，但它仍然主要基于相同的基本因素。当你输入一个单词时，你的键盘软件会问这三个问题:

1.  当“正确”输入时，哪些词典单词会产生与用户相似的点击模式？(你的点击构成了一个“键盘星座”，你的手机可以将它与字典中每个单词的理想星座相比较。)
2.  这些单词中哪一个更常用，以及*多常用*？(科希恩达指向 [Zipf 定律](https://simple.wikipedia.org/wiki/Zipf%27s_law) ，预测任何语言中的比例词频。)

3.  这些单词中的哪些会出现在用户的“动态词典”中——他们最常用的单词，或者他们联系人的名字，或者他们在“自动更正”更改后重新键入的单词？

对于每个单词，软件必须选择这三个因素的权重来决定你的意思。要理解这个选择有多难，风险有多大，记得花了多长时间教你的手机“操”这个词。

但是如果你不停下来修正每一个单词，这整个过程会更好。打一整句话或一条信息，然后在最后校对，效率会高得多。这就是该系统的设计目的，也是苹果用户测试中最有效的方式。

Kocienda 说，随着自动更正和预测文本变得越来越复杂，这一点将变得更加真实。随着键盘检查更长的段落，它们可以识别更多的错误。他举例说，你的手机可能会认为你输入了中的*，但几个词之后才意识到你本来是想输入 *if* 。(它知道你的意思不是“你喜欢 pia coladas，淋了雨……”)自动更正功能越先进，它就越能在无需您不断查看的情况下修复问题。所以要学会相信自动更正。(或者T5【关机】T6。)*

并且**不要一直依赖预测文本。**我经常使用这个坏习惯，检查键盘上方的建议单词，看看我是否可以点击完成的单词来节省几秒钟。柯辛达称之为 [上下文切换](https://en.wikipedia.org/wiki/Context_switch)——你在打字，然后你在阅读，然后你在看，每次你在这些任务之间切换，你都会损失一些时间和注意力。在苹果的测试中，大多数用户(不仅仅是高级电话打字员)如果只是完成自己的话，打字速度要快得多。除了不常见的单词或情况之外，最好忽略预测文本。

这种启示让我想起我是多么怀念黑莓和 Sidekick 上的硬件键盘。但是正如 Kocienda 指出的，因为软键盘可以消失，为两倍的显示空间腾出空间，所以它们对于全屏手机是必要的。在他看来，当我们拖着一个硬件键盘时，我们无法获得我们现在拥有的全功能应用程序。因此，我们必须放弃点击一个真正的物理按钮的简单性——这实际上并不那么容易。

Kocienda 告诉 Lifehacker，苹果与谷歌等竞争对手的区别在于，苹果总是旨在为大多数用户做出单一的最佳选择，而不是将选择权交给用户，这可能会让只想发送文本的忙碌人员不知所措。这意味着一些功能和选项在 iPhones 上出现的时间比在 Androids 上晚，但一旦它们出现，通常都是经过深思熟虑的设计。( [通常。](https://lifehacker.com/why-apples-iphone-7-headphones-dont-work-on-the-latest-1797826496#_ga=2.33531900.350626624.1536677154-1297080755.1497980211) )

柯辛达说，这是科技需要多样性的另一个原因:D:像 iPhone 这样的设备是为每个人设计的，如果你的设计和工程团队中没有每个人的参与，你就不可能为每个人设计。如果不考虑使用它的手和手指的大小，你就不能设计一个软件键盘。如果你在开始用户测试之前没有预见到问题，你就不能想出最好的解决方案。
