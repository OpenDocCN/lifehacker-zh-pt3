# 关于国会将加密技术定罪的最新尝试，你应该知道什么

> 原文:[https://life hacker . com/what-you-should-know-about-congress-latest-attempt-1770468594](https://lifehacker.com/what-you-should-know-about-congresss-latest-attempt-to-1770468594)

国会的一项新法案草案将迫使科技公司在执法部门要求的任何时候破坏或破解自己的安全功能和加密。听起来很可怕？确实是。这是账单上写的，以及你能做的。

对于那些刚刚了解情况的人来说，苹果和联邦调查局 [最近就圣伯纳迪诺大规模枪击案](https://lifehacker.com/how-the-fbi-s-fight-with-apple-could-change-the-future-1762392996) 枪手赛义德·里兹万·法鲁克的一部 iPhone进行了一场法律大战。联邦调查局要求苹果公司开发一种工具来绕过手机的 PIN 锁。苹果认为这是一个不必要的负担，会削弱所有 iPhones 的安全性。最终，联邦调查局做出了让步， [找到了一家第三方公司来解锁 iPhone](http://www.technobuffalo.com/2016/04/10/apple-were-not-going-to-sue-fbi-for-hacking-into-iphone/) ，尽管 [现在还有另一款手机](http://recode.net/2016/04/08/apple-fbi-encryption-battle-shifts-to-new-york/) ，只是在全国范围内。

然而，作为对整个事件的回应，参议员(D-CA)和 [理查德·伯尔](https://www.burr.senate.gov/) (R-NC)目前正在制定一项法案，以确保执法部门可以不必乞求就能得到他们需要的东西。范斯坦-伯尔法案如果获得通过，将迫使科技公司遵守法院命令，交出数据，即使这些数据是加密的，或者公司实际上无法访问这些数据。所谓的“2016 年遵守法院命令法案” [的初步版本上周五](https://www.scribd.com/doc/307378123/Burr-Encryption-Bill-Discussion-Draft) 发布。这个版本不一定是最终版本，但已经很糟糕了。除非做出重大改变，否则这项法案对任何重视自身安全的人来说都是危险的。

### **该法案将会做什么**

根据周五发布的草案，任何时候向科技公司提供信息的法院命令，他们都必须有能力遵守。要么通过访问数据本身，要么通过帮助政府找到访问数据的方法。换句话说，一个公司不能说“那不可能”就收工了。面对这样的订单，科技公司有两个选择:

*   **直接翻资料。**如果一家公司的服务器上有与法院命令相关的数据，他们将被要求将其移交给执法部门。它必须是“可理解的格式”这意味着公司必须有能力将加密数据转换成可读格式。这将要求提供加密的科技公司要么自己持有密钥来解密数据，从而使他们的客户数据更容易受到攻击，要么更糟糕的是，只使用公司自己可以破解的加密，从而使加密实际上一文不值。
*   帮助执法部门获取信息。如果一家公司没有将数据存储在某个地方，它将不得不提供“*必要的技术援助*，以帮助政府获取数据。换句话说，科技公司将被迫参与调查取证，直到政府决定这项工作已经完成。值得注意的是，这项法案并没有限制政府可以要求一家公司付出多少努力。然而，有一项条款规定，他们将因提供技术援助而发生的任何费用得到“补偿”。

以圣贝纳迪诺案件为例，根据这项新法律，苹果公司将被要求获得法鲁克的 iPhone，因为这是法院命令的主题，无论苹果公司认为这可能会损害他们的业务或客户的安全。然而，有些令人困惑的是，它故意没有说明苹果必须如何做到这一点。该法律的一节内容如下:

> 本法案中的任何内容均不得解释为授权任何政府官员要求或禁止任何相关实体采用任何特定设计或操作系统。

换句话说，联邦调查局不能带着绕过手机加密的特定软件功能的要求来找苹果(他们在圣贝纳迪诺案件中就是这么做的)。相反，它只是要求苹果必须以某种方式做这件事。它还说，苹果的工作不会完成，直到政府决定它已经完成。

该法案的范围还延伸到了应用商店。其中一节说，任何“为产品、服务、应用程序或软件分发许可证”的公司都必须确保这些产品能够遵守法律。换句话说，如果苹果不能确保应用程序开发者有能力交出其客户的数据，苹果就不能合法地允许他们的应用程序出现在应用程序商店中。同样，该法案没有说明公司应该如何确保他们分发的每一个应用程序都符合法院命令。充其量，它在法律上要求对商店中的每一个通信应用程序进行冗长的安全审计。在最坏的情况下，它需要应用程序商店的所有者来决定开发者可以使用哪些安全功能。不管你怎么解读，这都是不好的征兆。

### **一切*其他*这个账单有问题**

以目前的形式，这项法案对科技公司和消费者来说是灾难性的。最大的问题之一是，它所要求的在许多情况下可能实际上是不可能的。比如 WhatsApp 最近 [对所有消息](https://lifehacker.com/whatsapp-turns-on-end-to-end-encryption-for-every-messa-1769157642) 启用了端到端加密。谷歌在 Gmail 上也做了同样的事情。两家公司都无法在没有物理访问终端设备的情况下访问通过其服务发送的数据，并且任何传输中获得的数据肯定不是“可读格式”该法案将要求该公司找到一种方法，以执法部门可以读取和使用的方式移交这些数据，尽管这在*字面上*是不可能的。正如政策分析师朱利安·桑切斯所说，在某些情况下，这项法案无异于要求一家公司表演魔术:

 [https://lifehacker.com/embed/inset/iframe?id=twitter-718290682226601984&autosize=1](https://lifehacker.com/embed/inset/iframe?id=twitter-718290682226601984&autosize=1) 

WhatsApp 会有两个选择。一方面，他们可以禁用端到端加密，这使得他们的产品本质上不太安全，并扰乱了他们的客户群。或者，他们可以建立一个后门或维护一个客户加密密钥的数据库，这 [破坏了平台的安全](http://searchsecurity.techtarget.com/definition/back-door) 。这就像要求建造你房子的人拥有一套你家的钥匙，并且有一扇只有他们才能进入的特殊的门。

这两种选择都削弱了消费者的安全性，并为其他坏人打开了大门，他们可能会窃取用户数据、消息和通过应用程序发送的任何其他信息。根据这项法律，**强有力的安全措施将是非法的**。如果产品或服务非常安全，公司或政府无法访问或解密，公司将不得不削弱这种安全性，以遵守法律。

另一个主要问题是，该法案要求苹果、谷歌和微软等公司监管其应用商店中的安全应用并删除它们。这些公司不仅要削弱自己产品的安全性，还要确保他们应用商店中的任何应用程序也有薄弱的安全性。除了禁止安全应用，这将给应用开发者和应用商店带来额外的负担，以确保每个应用都遵守这项法律。

该法案甚至可能没有必要。(我们在报道苹果/联邦调查局案件 时提到过 [)允许法院命令一家公司以任何必要的方式协助调查，只要合规不是不合理的负担。在苹果与联邦调查局的斗争中，苹果辩称，创造联邦调查局想要的工具是一种不合理的负担，会危及更多 iPhones 的安全，而不仅仅是本案中的一部。但是苹果在](https://lifehacker.com/how-the-fbi-s-fight-with-apple-could-change-the-future-1762392996) 之前已经多次帮助执法部门在不同情况下 [提取其他 iPhones 的数据。新法案将危及世界上每一个设备和应用的安全，仅仅是为了处理一些异常情况，在这些情况下，政府无法利用现有法律——或者，正如联邦调查局在圣贝纳迪诺案件中证明的那样，现有的安全研究人员和承包商——获得他们需要的信息。](http://techcrunch.com/2016/02/18/no-apple-has-not-unlocked-70-iphones-for-law-enforcement/)

该法案的支持者表示，有必要解决“变暗”问题。随着安全技术越来越好，执法工作变得越来越难。过去，高级加密和安全层是政府和高度组织化的犯罪分子的专属。现在，每个拥有最新智能手机的人都有可能阻挠联邦调查。这给执法部门带来了跟上技术先进的罪犯的巨大且日益增长的负担。然而，正如我们的总编辑艾伦·亨利解释的那样，事情本该如此。

> 联邦调查局、国家安全局和中央情报局不应该爬到硅谷来破解电话或加密。他们应该已经有能力这么做了(T2)，如果没有，他们还在等什么，他们在过去的 20 年里去了哪里？

虽然“变暗”是一个合理的问题，但招募制造我们所有小工具和应用程序的科技公司是一个糟糕的解决方案。执法机构应该配备他们进行调查所需的工具，而不损害没有做错任何事情的用户的安全。每个美国公民都不应该在他们的前门上留一把不牢固的锁，以防有一天政府需要把门撞开。

### **如何在这个问题上发出自己的声音**

目前，该法案的最终版本还没有正式发布，所以还有足够的时间来发表意见。你可以在这里找到如何联系参议员伯尔的办公室的信息和参议员费恩斯坦的办公室的信息。您也可以 [使用 4USXUS](https://lifehacker.com/4usxus-is-your-one-stop-shop-for-keeping-up-with-your-c-1758949064) 查找您的参议员和众议员的信息并获取联系信息。你也可以 [使用 Democracy.io 轻松联系他们](http://lifehacker.com/democracy-io-makes-it-dead-simple-to-contact-members-of-1719265246) 无需搜寻联系信息。使用这两种方法中的任何一种，让你的代表知道你对该法案的看法，以及你认为他们应该如何投票，如果该法案到达委员会，或者上帝保佑，全票通过的话。你也可以在这里 [联系白宫](https://www.whitehouse.gov/contact/mobile) 让总统知道，如果法案通过，你支持否决。Gizmodo 也有一份关于 [当前竞选总统的每个总统候选人](http://gizmodo.com/the-2016-presidential-candidates-views-on-computer-surv-1760871747) 的计算机安全观点的综述。

一旦法案正式出台，你也可以使用 4USXUS 来关注法案本身(例如， [这是 CISA 的样子](https://4usxus.com/Bills/Details/s-754_congress114) )。与此同时， [向你的国会代表伸出援手](http://lifehacker.com/how-to-contact-and-properly-communicate-with-your-gover-5871810) 让他们听到你的声音总是一个好时机。现在，这个法案似乎不太可能通过(它已经引起了很多负面的关注)。然而，更糟糕的法案已经在没人注意的情况下通过了，所以尽可能花时间让别人听到你的意见。
