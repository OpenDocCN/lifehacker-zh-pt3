周一彭博新闻 [投下一枚重磅炸弹](https://www.bloomberg.com/news/articles/2018-04-02/apple-is-said-to-plan-move-from-intel-to-own-mac-chips-from-2020?utm_campaign=socialflow-organic&utm_content=business&utm_source=twitter&utm_medium=social&cmpid=socialflow-twitter-business) 报道。到 2020 年，它声称苹果将停止在其电脑中使用英特尔 CPU。想象一下:世界第三大个人电脑制造商有一天可能会超越最大的电脑芯片制造商。

消息传出后， [英特尔的股票很快](http://fortune.com/2018/04/02/why-apple-dump-intel/) 下跌，虽然推特上的很多人兴奋不已，但其他人则更加警惕，甚至担忧。相对而言，苹果制造自己的 CPU 芯片是史无前例的，这一举动可能意味着许多事情。因此，让我们深入了解实际发生了什么，以及这一切可能意味着什么。

### 谣言

苹果和英特尔分手的谣言相当普遍，正如 Moor Insights & Strategy 的总裁兼首席分析师 Patrick Moorhead 昨天在 Twitter 上指出的那样。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-980899814392107009&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980899814392107009&autosize=1) 

似乎每次苹果和英特尔来到谈判桌前，谣言就会爆发——正如穆尔黑德指出的那样，媒体在这个过程中被用作工具 。仅在最近两年类似的传闻就不断出现在 [最少](https://www.theregister.co.uk/2017/02/01/apple_weaning_off_of_intel/) [四个](http://fortune.com/2016/06/18/apple-intel-deal-moores-law/) [单独](http://vrworld.com/2016/10/24/apple-to-abandon-intel-chips-for-future-macs/) [的场合](https://www.macrumors.com/2017/09/29/apple-interested-processors-modems/) 。

然而，这里的不同是最新谣言的来源。彭博作家马克·古尔曼有保护苹果泄密的历史，苹果是一家以保密 而闻名的 [公司。](https://theoutline.com/post/1766/leaked-recording-inside-apple-s-global-war-on-leakers?zd=1&zi=qjswshwz)

理论上，这可能只是苹果公司利用一名记者在谈判中采取强硬手段，但让我们假设古尔曼不是苹果公司的棋子，这些信息是合法的。根据古尔曼与彭博记者伊恩·金共同撰写的文章，苹果公司内部有一个新的倡议，称为 Kalamata，Kalamata 的目标是到 2020 年将苹果设计和制造的芯片放在 MAC 电脑中。

谣言之外的一切都是记者和评论员对谣言的进一步推测。

但是自从彭博揭露了这个故事，关于可能会发生什么的推测和理论已经满天飞，所以让我们试着更好地理解这些推测。

### 场景一:苹果开始在所有电脑中使用基于 ARM 的 CPU

这绝对是现有的最流行的理论，彭博和边缘的都提出了这个理论。在这种情况下，苹果不只是告别英特尔；它告别了英特尔芯片运行的 x86 架构，转而拥抱 ARM，这是 iPad 和 iPhone 中的 CPU 所基于的。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-980928217123975168&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980928217123975168&autosize=1)  [https://gizmodo.com/embed/inset/iframe?id=twitter-980880988388646912&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980880988388646912&autosize=1)  [https://gizmodo.com/embed/inset/iframe?id=twitter-980903374685077504&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980903374685077504&autosize=1)  [https://gizmodo.com/embed/inset/iframe?id=twitter-981224625999941633&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-981224625999941633&autosize=1) 

有一种理论认为，苹果非常擅长制造超快速高效的手机 CPU，而英特尔最近停滞不前——三年多来未能提出一项重大且必要的 CPU 重新设计。因此，尽管苹果在移动领域的规模和成功仍然没有计算机处理器方面的背景，但理论上它可以做世界上最大的计算机处理器生产商做不到的事情。

现在，如果有人能在这方面取得成功，那肯定是苹果。该公司在计算机处理器技术方面的不足，在资源方面得到了弥补。见鬼，该公司可能会收购英特尔或其竞争对手 AMD，并且仍有剩余资金收购其他公司 [每个人都认为苹果应该收购](https://gizmodo.com/fools-wont-stop-pontificating-about-what-apple-needs-to-1795087997) 。

但这绝不容易。[Errata Security 的 Robert Graham 昨天写了一整篇 Twitter 帖子](https://twitter.com/ErrataRob/status/980907462973902849) 讲述了人们购买 iMac Pros 和 MacBook Pros 时使用 ARM 处理器进行高端计算的问题。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-980907462973902849&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980907462973902849&autosize=1) 

在这篇文章中，Graham 指出，ARM 处理器以其能效而闻名，因为它们实际上并不打算像几乎所有主要笔记本电脑、台式机和服务器中的 x86 处理器一样快。它可能适用于 Chromebook 或 iPad，或者 Cloudflare 服务器群的非常特殊的需求，但它不一定能很好地满足某人渲染几个小时的 4K 镜头或甚至几分钟 Adobe After Effects 制作的东西的需求。

还有另一个大问题:macOS 是为 x86 架构设计的。转移到 ARM 将需要对软件的工作方式进行重大的重新设计，这意味着最终对 x86 计算机的支持将会结束，这正是苹果上次在 2006 年进行重大架构变革时发生的情况。当时，它从摩托罗拉的 PowerPC 转移到了英特尔。上一个支持 PowerPC 的操作系统是 2007 年 10 月发布的 Mac OS X Leopard——就在移动开始一年后。

除了对 macOS 传统支持的终结，从 x86 到 ARM 的转变也会影响软件开发人员。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-980954583198552069&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980954583198552069&autosize=1) 

对于那些主要使用苹果自己的软件语言的开发人员来说，这一举措可能是相对无缝的。但它可能会对软件巨头造成更大冲击。像 Adobe 这样的公司经常使用代码来利用设备的 CPU 甚至 GPU。他们的软件目前针对 x86 架构进行了优化，可能需要一些重要的编辑才能在 ARM 上工作。

如果你认为软件的这些变化会很快出现，请三思。像 Adobe、微软和 Autodesk 这样的公司都已经在努力在 Mac 和 Windows 上提供完全相同的体验——加入架构变化可能会使这变得更加困难。

看看谷歌的 Chrome OS 就知道了。它在教育部门非常受欢迎，但它的用户仍然无法访问 Adobe 的完整桌面软件，iPad 用户也无法访问，尽管苹果继续将平板电脑作为 Chromebook 的竞争对手 。

还要看看苹果和 AdobeT3 之间的 [多灾多难的历史。除了在 iPhones 上争夺 Flash 之外，两家公司还为 Adobe 在 Mac OS X 的支持争吵不休，在 2003 年的某个时候，](https://www.cnet.com/news/adobe-and-apple-allies-and-rivals-through-the-ages/) [Adobe 甚至完全取消了对 Premiere](https://www.cnet.com/news/jobs-why-apple-banned-flash-from-the-iphone/) 的支持，因为这将需要更多的工作，而公司认为这在财务上是不可行的。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-981017723936362496&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-981017723936362496&autosize=1) 

全面转向 ARM 的成长之痛可能会很严重，疏远让苹果维持了几十年的市场——创意市场——的可能性会很大。如果苹果的 ARM 处理器无法处理创意人员所需的繁重工作负载，他们就会停止投资苹果产品，这也可能导致软件公司停止投资。

但如果有一家公司能吓唬我们，让我们爱上一款更符合 iOS 的新 macOS，其功能和速度不如前代(在比任何装有 Windows 的机器都慢的机器上)的应用程序，那就是苹果。

只是也许强迫每个人的手臂并强迫他们进入一个新的生态系统并不是实际的计划——尽管彭博的古尔曼已经多次报道了苹果公司进一步整合苹果硬件和软件的计划 并转向某种更统一的操作系统。也许苹果没有大的转变，而是计划了某种桥接设备？嫁给两个世界。

### 场景二:苹果开始只在部分电脑中使用基于 ARM 的 CPU

为了继续迎合比如视频编辑和插画师的高功率需求，苹果可以决定不将其所有计算机从 x86 上移走。相反，它只能将价格更低的电脑转移到 ARM 上。这意味着要么是全新的 MacBook 或 MacBook Air——目前市面上最便宜的两种苹果电脑——要么是全新的东西，正如前《华尔街日报》专栏作家兼 the Verge 和 Recode 编辑沃尔特·莫斯伯格所建议的。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-980883911344615424&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-980883911344615424&autosize=1) 

从很多方面来说，这感觉更可行。苹果不会试图扭曲 ARM，或者彻底重新设计 macOS，并要求主要软件合作伙伴设计新的架构，而是会推出基于自己芯片的廉价笔记本电脑，这也会让人们更习惯于 macOS 和 iOS 在未来某一天融合的想法。预示苹果新生态系统的新机。

在短期内，这种设备似乎很有可能，但我们不知道操作系统会是什么样子。iOS 将是有意义的，因为它已经被设计为在 ARM 处理器上运行。试图强迫 macOS 使用它没有什么意义。今年早些时候，微软与生产基于 ARM 的移动 CPU 的高通和生产笔记本电脑的华硕合作，发布了一款采用高通骁龙处理器的 Windows 10 笔记本电脑。 [电池寿命不可思议](https://gizmodo.com/the-battery-life-is-so-good-ill-forgive-windows-on-snap-1823906297#_ga=2.84953754.2028028609.1522671452-3494214420.1521481782)——速度明显不如。

在目前的状态下，将 macOS 放在这样一个设备上，即使里面有苹果的伟大 CPU，也可能不会有好结果。但也许苹果不会真的将 ARM 作为其所有设备的架构。

### 场景三:苹果启动英特尔，但坚持使用该架构

在这种情况下，苹果停止使用英特尔 CPU，但继续使用 x86 架构。这意味着 macOS 或你最喜欢的软件不会有大的根本性改变，也不会有失去像 iMac Pro 这样功能强大的电脑的威胁。理想对吗？

只是，据我们所知，苹果还没有准备好开始设计 x86 CPUs。这使得 [没有公开的重大收购或知识产权许可](https://twitter.com/SWenerski/status/981209173085048832) 表明它有团队来制造这些芯片，而且似乎没有任何内部设计师在 x86 领域的工作特别出名。

他们真的需要一个懂 x86 的伙伴来让它工作。

### 场景四:苹果启动英特尔，转而开始与 AMD 合作

这是真实的月球拍摄场景；苹果停止购买英特尔 CPU，转而开始与 AMD 合作基于 x86 架构的半定制芯片。

 [https://gizmodo.com/embed/inset/iframe?id=twitter-981201120411377664&autosize=1](https://gizmodo.com/embed/inset/iframe?id=twitter-981201120411377664&autosize=1) 

对于不希望苹果远离 x86 的人来说，这很有意义。AMD 有一个非常积极的未来几年发布 CPU 和 GPU 的路线图，它目前的 CPU 和 GPU 受到了好评，它已经将其技术授权给其他人，包括索尼(Playstation 4)、微软(Xbox One)和英特尔(新 G 系列处理器[)。](https://gizmodo.com/rivals-intel-and-amd-got-together-and-made-something-be-1824178636)

苹果可以与 AMD 达成协议，在相当短的时间内开始生产超快的芯片。但这一点，以及包括保留 x86 架构在内的任何场景，似乎都与苹果希望统一其软件和硬件的传言背道而驰。与 AMD 联手可能会在短期内帮助苹果，肯定会让它更好地理解如何为笔记本电脑、台式机和服务器设计 CPU，但这不一定会让苹果走向任何形式的大统一。

苹果实际会怎么做还有待观察。但是无论你怎么看，CPU 空间都将变得更加有趣。
