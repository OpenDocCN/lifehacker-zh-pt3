### Raspbian 是最好的全能操作系统

Raspbian 是树莓派的“官方”操作系统，正因为如此，它是大多数人想要开始使用的操作系统。

Raspbian 是专门为 Raspberry Pi 构建的 Linux 版本。它装有你用电脑完成每项基本任务所需的所有软件。你将获得 LibreOffice 办公套件、网络浏览器、电子邮件程序和一些向儿童和成人教授编程的工具。见鬼，它甚至包括一个 [的特殊(不再开发)版本的《我的世界》](http://minecraft.gamepedia.com/Pi_Edition) 。拉斯边是很多 DIY 项目的骨干，所以如果 [你想做点什么](http://lifehacker.com/ten-more-awesome-projects-for-your-raspberry-pi-5978871) ，拉斯边很可能是你想开始的地方。因为它的应用非常广泛，所以也很容易找到指南和故障排除技巧。

如果你是 Linux 新手，Raspbian 可能会让你有点困惑。[Rpi 初学者维基](http://elinux.org/RPi_Beginners) 是一个很好的起点， [官方树莓 pi 资源](https://www.raspberrypi.org/resources/) 也是如此。两者都带你了解使用 Raspbian 所需的一切，从制作游戏到掌握默认程序。如果你更喜欢视频，那么 YouTube 频道的 [树莓派新手版](https://www.youtube.com/user/RaspberryPiBeginners) 有很多可以看，还有 [树莓派教程](https://www.youtube.com/user/RaspberryPiTutorials)[C](https://www.youtube.com/user/RaspberryPiTutorials)h[a](https://www.youtube.com/user/RaspberryPiTutorials)n[n](https://www.youtube.com/user/RaspberryPiTutorials)e[l](https://www.youtube.com/user/RaspberryPiTutorials)。

### OSMC 是最好的，功能最丰富的媒体中心软件

[【OSMC】](https://osmc.tv/)(开源媒体中心)是基于 [Kodi](https://kodi.tv/) (原 XBMC)的媒体中心软件，但是更容易设置和使用。其实一点也不像 Kodi，这是好事。这可能是 Pi 上最容易使用的媒体中心软件。如果你是媒体中心的新手，或者你正试图为非技术人员建立一个媒体中心，OSMC 是你想要使用的。

与 Kodi 相比，OSMC 有一个干净的界面，去掉了很多废话。你会在屏幕左侧看到一个菜单，让你选择媒体(视频/音乐/图片)，深入设置，或检查其他程序。一切都很整洁，不言自明。当然，你仍然可以为媒体流安装 Kodi 插件 和 [设置遥控器](http://lifehacker.com/enhance-your-xbmc-experience-with-remote-controls-for-a-5501010) ，这样你就不必使用键盘了。事实上， [OSMC 已经预置了几个流行的遥控器](https://osmc.tv/wiki/general/configuring-a-remote/) ，所以你甚至不需要绞尽脑汁去设置一个。至于本地媒体，您可以播放 USB 存储中的视频和照片。

尽管如此，OSMC 仍有很大的修补空间。它运行完整版的 Debian，所以如果你是高级用户，你可以设置 SSH、FTP、Samba 共享和 [等等。](http://lifehacker.com/how-to-automatically-download-virtually-anything-as-soo-5943163)T3】

### OpenELEC 是一个媒体中心，面向那些想要简单快捷体验的人

如果 OSMC 不适合你的需求， [OpenELEC](http://openelec.tv/) (开放式嵌入式 Linux 娱乐中心)值得一看。OpenELEC 是 Kodi 的一个更直接的端口，所以如果你熟悉 Kodi 及其工作方式，你就可以轻松自如了。OSMC 是一个功能丰富、可定制的媒体中心，可以做你想做的任何事情，而 OpenELEC 只为一件事而生:播放媒体。如果你的硬盘上已经有了大量的电影或音乐，而你只是想用一种简单的方式在电视上播放它们，OpenELEC 是一个不错的选择。

在 之前我们已经深入研究过 OpenELEC，但是它主要的吸引力在于它的速度。OpenELEC 采用 Kodi，去掉了许多定制选项，以保持它的简洁、快速和简单。然而，它不像 OSMC 那样开放，所以你不能在不钻研复杂菜单的情况下改变系统级别，比如改变 Pi 的超频速度。OpenELEC 还限制对某些服务的访问，比如 SSH，所以设置起来并不容易。

### Windows 10 物联网核心面向开发互联设备的开发者

[Windows 10 IoT](https://ms-iot.github.io/content/en-US/Downloads.htm) 是为树莓派打造的 Windows 特别版。这是 Windows 的完整版本*而不是*。相反，它是一个开发平台，供编码人员和程序员使用 Raspberry Pi 和 Windows 10 开发互联网连接设备的原型。Windows 10 物联网只与 Windows 10 兼容，除非你有另一台安装了 Windows 10 的电脑，否则你不能用它做任何事情。

当你第一次启动 Windows 10 IoT 时，你在 Pi 上看到的只是上面的屏幕。您不能单独控制 Pi 或在 Pi 上做任何事情。为此，你需要 [下载并在你的 Windows 电脑上安装 Visual Studio](https://developer.microsoft.com/en-us/windows/iot/Downloads.htm) 。一旦你这样做了，你就可以在 Windows 10 的 Visual Studio 中编程和控制你的 Raspberry Pi 了。这意味着你可以触发闪烁的灯，连接到按钮，控制电机和无数其他东西。

为了开始使用 Windows 物联网核心，微软有一个 [奇妙的项目集合](https://microsoft.hackster.io/en-US) 教你如何使用它。看看这些项目，看看是否有你感兴趣的，以决定 Windows 10 物联网核心是否值得安装。

### RISC 操作系统适合任何想玩完全不同类型的操作系统的人

如果这里有一只奇怪的鸭子，那就是 [RISC OS](https://www.riscosopen.org/wiki/documentation/show/Welcome%20to%20RISC%20OS%20Pi) 。RISC OS 不是建立在 Linux 上的，也不是真正帮助电子修补匠制造的原型。相反，它是一个完全独立的操作系统。它也很奇怪，但是玩起来会很有趣。

RISC 操作系统与其他操作系统如 Linux、OS X 甚至 Windows 没有太多共同点。它最初设计于 1987 年，起源于英国广播公司的。RISC OS 比现代操作系统简单得多。单个应用程序可以接管整个操作系统，它只能作为单用户系统工作，应用程序只是名称前面带感叹号的目录，它在安全性方面没有太大的作用。RISC 操作系统也非常擅长拖放，如果你想保存，你可以把一个“另存为”图标拖到一个文件夹中。基本上，这是一个令人困惑的小操作系统，但它仍然令人着迷。

大多数人不应该安装 RISC OS。你不能真的把它作为一个主要的操作系统，它没有太多的现代软件支持，也不像现在其他可用的操作系统那样工作。也就是说，如果你是好奇型的，玩这个很有趣。对于初学者指南， [Ident Showcase 在 YouTube](https://www.youtube.com/watch?v=qNhkHyWRwg0) 上有一个很好的演示， [RISC OS 欢迎页面会引导您了解一些基础知识](https://www.riscosopen.org/wiki/documentation/show/Introduction%20to%20RISC%20OS) ，或者 [查看 RISC OS 论坛以获得提示](https://www.riscosopen.org/forum/) 。
