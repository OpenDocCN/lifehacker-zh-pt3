# 向孩子们解释编码，让它变得可理解

> 原文：<https://lifehacker.com/to-explain-coding-to-kids-make-it-relatable-1797506114>

孩子们有一种快速获取信息的方式(有时比我们想象的要快)...)，但当解释更抽象的概念，如编码或计算机编程时，你可能需要的不仅仅是讲座或一些快速的 YouTube 解释器。这就是程序员托梅克·卡扎诺夫斯基 在向一群 6 岁的孩子解释这项技能时学到的东西，其中就有他自己的女儿。



为了确保他理解了关于编程的信息，他想出了几个简单的策略，让向孩子们解释概念变得容易，让老师和学生都参与进来。

### 从基础开始

从头开始会有帮助。Kaczanowski 以询问这群孩子是否知道什么是计算机程序员和计算机开始了他的解释。

他用简化的解释来解释计算机如何处理数据输入和输出，用伪代码来解释编程语言。从那里，他能够讨论什么是合格的计算机，以及程序员如何与笔记本电脑、汽车和智能手表等计算机进行交互。

### 问孩子问题

孩子们喜欢问和回答 的问题，问这些问题是了解他们所知的一个很好的方式。它还提供了纠正他们可能有的任何潜在的令人困惑的误解的机会。

Kaczanowski 向小组成员询问了有关计算机、数据库和编程语言的问题。

从那里，他能够收集他的观众知道什么，解释概念，并转移到下一个主题。

### 让它变得有关联

你的孩子可能不理解数据库或处理器的概念，但他们知道玩具箱或书架是如何工作的，以及大脑是干什么用的。

因此，当解释像编程这样的概念时，让它与他们已经知道的东西相关联是非常有帮助的。在这种情况下，Kaczanowski 提到了一个他的观众非常熟悉的网站:YouTube。“我转到 YouTube，因为我知道他们会知道很多，”Kaczanowski 说。“我从画他们都很熟悉的 UI 开始。”

熟悉的卡通模型帮助他解释了数据库的工作原理，同时使用 Peppa Pig 等例子让孩子们思考视频到底来自哪里，以及 YouTube 如何知道播放哪些视频。参考他们日常生活中的物体——比如大脑、书架和 YouTube——有助于将抽象概念与具体主题联系起来。

### 清晰并重复

你在这里和孩子们打交道，所以你应该 [期望自己](http://lifehacker.com/teach-your-kids-to-stop-interrupting-you-with-this-tech-1773363708) 重复好几次。Kaczanowski 借助简单的图片，比如网页模型和他解释时画的一叠 DVD，他可以反复参考。他写道:“这抓住了他们的注意力，让我指着一幅画，提醒他们我们几分钟前讨论过的事情。”图画和重复的陈述可能看起来很乏味，但 Kaczanowski 发现像突击测验一样使用图像有助于孩子们记住术语和概念。
