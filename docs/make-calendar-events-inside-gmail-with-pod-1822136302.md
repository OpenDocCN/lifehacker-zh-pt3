# 使用 Pod 在 Gmail 中制作日历事件

> 原文：<https://lifehacker.com/make-calendar-events-inside-gmail-with-pod-1822136302>

Gmail 和谷歌日历的整合令人尴尬。Gmail 永远无法识别所有重要的活动信息，为了解决这个问题，你必须在一个新的标签页中打开谷歌日历，这违背了它的初衷。值得庆幸的是，有第三方修复，如 Gmail 的 [Pod，它可以识别事件信息，并将其加载到 Gmail 窗口内的完整日历事件中。](https://gsuite.google.com/u/1/marketplace/app/pod_for_gmail/93796446482?pageId=none)



[Pod 的介绍文章](https://pod.io/blog/pod-for-gmail.html) 展示了一个完美的用例，但 Gmail 的 Pod 对我来说并不总是很好。它很容易地创建了一个事件，但是当我运行更多的演示来获得上面的 screencap 时，它开始出错。不过，Pod 在正确的轨道上加载了我现有的时间表，我可以在手边有电子邮件的情况下输入所有的活动信息。因此，即使在故障状态下，Pod 也能让从电子邮件到日历的转换压力更小，更不容易出错。

该插件来自 iOS 日历应用程序 [Pod](https://pod.io/) 的制造商。我总是对承诺“重新思考”联系人、日历和信息等概念的应用保持警惕——我已经被太多雄心勃勃的应用烧伤了——但我喜欢任何能减少三者之间摩擦的东西。
