### 竞争者

BitTorrent 客户端的世界是广阔而无限的，但在能够下载 Torrent、安排下载、远程管理下载等最受欢迎的应用程序中，有三个应用程序非常明显:

*   [**【Torrent(Windows/Mac/Linux)**](http://www.utorrent.com/):Torrent 一直以来都是你最喜欢的 BitTorrent 客户端。它是跨平台的、免费的、易于使用的，并且包含了许多有用的特性。像任何优秀的 torrenting 客户端一样，它可以恢复停止的下载，合并跟踪器，按顺序下载项目，支持加密文件，并让你通过移动应用程序远程管理下载。它可以安排下载，支持端口转发，甚至可以根据您的整体带宽使用情况自行调节。此外，它是跨平台的，无论您是否需要高级功能，使用起来都很简单。
*   [**【qBittorrent(Windows/Mac/Linux)**](http://www.qbittorrent.org/):免费、开源，旨在重拾 torrent 留下的社区，qBittorrent 因其纤薄、简洁、超快而获得了大量追随者，同时也没有忽略重要的功能。它的界面可能看起来稀疏，但在引擎盖下，你会发现你需要的一切，无论你是重型还是轻型下载器。IP 过滤、顺序下载、内置搜索、加密下载、基于 web 的远程控制、端口转发，一切尽在其中。它也是跨平台的，几乎可以在任何系统上完美运行。
*   [**【Transmission(Windows/Mac/Linux)**](http://www.transmissionbt.com/):Transmission 出现在这篇综述中是因为一个不幸的原因(后面会有更多的介绍)，也是因为我们过去经常推荐它——而且理由很充分。它是轻量级的，并且在很长一段时间里是 Mac 上最好的烘焙应用。从那以后，它就跨平台了，第一方和第三方的版本都可以在 macOS、Linux 甚至 Windows 上使用。像我们的其他竞争者一样，它支持加密、共享您自己的文件、IP 过滤、下载调度和远程管理。然而，与我们这里的一些其他产品不同，它为 NAS 设备、家庭服务器、HTPCs 甚至 Raspberry Pis 提供了一个杀手级的无头模式，并且它可以定制为 [完全自动下载](https://lifehacker.com/how-to-automatically-download-virtually-anything-as-soo-5943163) 。它最近经历了一些困难，但它仍然是一个强有力的竞争者。

这些特殊的客户只是冰山一角。它们是最受欢迎的，但其他的，如(Win/Mac/Linux)和[Vuze](http://www.vuze.com/)(Win/Mac/Linux)也是很好的竞争者。我们只是不可能在这样的摊牌中对他们进行比较。也许我们会让他们与这里的赢家对抗，或者在未来的摊牌中彼此对抗，如果有兴趣的话。尽管这里的底线是有很多选择，即使你不喜欢这三个——如果你不喜欢我们到目前为止提到的任何一个选项，还有更多选择。

### qBittorrent 提供了最好的 torrent 而没有膨胀

在某种程度上，qBittorrent 被特别设计成“反 torrent”。它的特性集非常相似，并且该项目是完全开源的，所以如果您愿意，您可以检查版本和修订版之间的变化，并自己开发应用程序，为项目做出自己的贡献。也就是说，我们大多数人不会，但很高兴知道有一个社区关注着项目幕后发生的事情。

尽管 qBittorrent 是专门为 torrent 设计的替代品，但它不仅仅是一个克隆。qBittorrent 独树一帜，其功能包括用于完成下载的电子邮件、基于网络的远程管理(HTPCs 或家庭服务器的理想选择)、内置搜索，这样你就不必处理不良信息(而且 [经常更换或关闭](https://gizmodo.com/kickass-torrents-clones-are-going-to-keep-appearing-ar-1784165878) )公共追踪器、隐私 IP 过滤、端口转发，这样你就不必使用默认设置、带宽调度，这样你在玩游戏或播放电影或音乐时就不会下载，甚至还有一个 torrent 创建工具，这样你就可以使用 [共享自己的大文件](https://lifehacker.com/how-to-share-your-own-files-using-bittorrent-5534190)

当然，虽然 qBittorrent 是快速、轻量级和跨平台的，但它在一些平台上比其他平台更出色(稍后会详细介绍)。)它与 Torrent 相比很好，因为这是它被设计来取代的，但 qBittorrent 更难与 Transmission 这样的应用程序相比。它不像 Transmission 那样具有开箱即用的无头功能，Transmission 可以配置为自动从文件夹、RSS 提要和其他来源下载，无需您的交互，这使得它非常适合于 [在](https://lifehacker.com/how-to-automatically-download-virtually-anything-as-soo-5943163) 可用时下载您想要的任何内容。它还具有命令行功能，可以在没有 GUI 的 linux 上运行，或者远程管理——如果您计划在 Raspberry Pi、家庭影院 PC 或网络连接存储(NAS)设备上运行它，这几乎是必需的。这并不是说 qBittorrent 不能被 [拼凑成一个类似的解决方案](https://github.com/qbittorrent/qBittorrent/wiki) ，只是传输要容易得多。

### 您的操作系统很重要，但不如您的用例重要

当我们说“您的操作系统很重要”时，我们真正谈论的是您的使用情形。当你需要一些东西时，你偶尔会从 Windows PC 上下载吗？或者当你睡觉或工作时，你会运行一个为你下载大量数据的家庭服务器吗？你是 Mac 用户，还是 Linux 用户？所有这些事情都很重要，但只是在你需要你的下载解决方案有多复杂的程度上。

Transmission 原本是为 MAC 和 Linux 系统设计的(让我们听听 GTK+，)，带有 Transmission-QT 和一个为 Windows 用户设计的 [半官方版本](http://lifehacker.com/transmission-bittorrent-client-finally-arrives-on-windo-1767688106) 。运行自己的 seedbox，还是在新的家庭服务器上安装 Linux？传输可能已经存在，或者几个按键就可以了，你可以肯定它会很好地与你的系统配合。就 Torrent 而言，它可能拥有最广泛的客户群，拥有适用于 macOS、Windows、Linux 和 Android 的原生应用程序，但它的功能也不会倾向于无头下载。qBittorrent 是跨平台的，在任何地方都能很好地工作，但它的开发重点是跨平台和易于管理( [基于 Qt](https://www.qt.io/developers/) ，)，而不是围绕任何特定平台的本机客户端。

所有这些归结起来就是你是否关心用户界面，或者你需要你的 torrenting 应用程序是一个原生的。如果你是一个偶然的下载者，你可能不会。如果你运行自己的 seedbox，而你的私人追踪器上的同行依赖于你，或者你让你的服务器做下载，并且你想要一些坚如磐石的可靠的东西，而不需要投入和排除故障，你可能会这样做。在决定之前，请考虑您的使用案例和操作系统。也许极客，超级定制，无头的传播诱惑太强，无法抗拒。也许你只是偶尔下载——在这种情况下 qBittorrent 或 torrent 就可以了。

### qBitTorrent 最精简，Transmission 最先进，Torrent 不错但是很难推荐

如果你是一个临时用户，需要的时候下载，想要的时候播种，我们推荐 qBitTorrent。它是轻量级的，易于安装，跨平台，超级快。如果你能设置的话，基于网络的远程管理是很棒的，它的特性足以让高级用户在他们想要的时候下载他们想要的东西。

如果你喜欢开放 API，有一个家庭服务器，大量下载所有东西，并随后播种大量文件，假设你使用 Mac 或者在基于 Raspberry Pi 或 Linux 的系统上运行，那么传输对于你的用例来说是理想的。见鬼，Transmission 有专门针对您正在运行的 NAS 或 HTPC 等嵌入式系统的版本，尽管最近受到恶意软件的威胁，它仍然是可靠的。它的开发团队真的需要赢回用户社区的信任，这是一件困难的事情，因为同一个社区也(理所当然地)关注隐私、匿名、安全，并且完全基于信任。

最后，Torrent 可能是“标准”，因为很多人使用它，但很难推荐给那些正在寻找新的 torrenting 客户端的人。当然，这很有效，而且效果很好。它有其他两个没有的功能和移动管理应用程序。它的用户界面时尚，易于使用，易于导航，甚至与旨在复制这种体验的 qBittorrent 相比也是如此。然而，告诉某人使用一个应用程序是很难的，但*也*确保拒绝所有垃圾软件， [禁用广告](https://lifehacker.com/disable-ads-in-utorrent-via-settings-826283231) ，确保每个新版本都不会带有一些阴险的东西(或者等到社区发现它是否存在后再安装)，等等。除非你对你现有的版本完全满意，或者你需要的某个功能在别处找不到，否则你可能会想放弃它——如果你正在寻找新的客户，你有更快、更轻便的选择。

尽管如此，我们还是不能替你选择。最终，这三者都有其优点，同样，这些不是唯一值得考虑的。也许这些都不适合你，你应该看看，或者，或者 [Tixati](http://www.tixati.com/) ，或者[r current](http://libtorrent.rakshasa.no/)。底线是始终考虑您的用例和需求，而不仅仅是流行什么，并首先选择适合它们的东西。
