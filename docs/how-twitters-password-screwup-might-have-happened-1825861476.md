# Twitter 的密码错误是如何发生的

> 原文：<https://lifehacker.com/how-twitters-password-screwup-might-have-happened-1825861476>

上周，Twitter 透露其曾 [不小心以明文](https://lifehacker.com/change-your-twitter-password-now-1825756672#_ga=2.121511624.1512506225.1525700096-1297080755.1497980211) 的形式存储了一些用户密码，并因此建议所有用户更改其 Twitter 密码。很糟糕。但据企业云提供商 BoltonSmith 的创始人特里斯坦博尔顿(Tristan Bolton)称，老实说，情况并没有那么糟。我们和他谈了这件事可能是如何发生的，以及它可能会变得更糟。



### **它通常是如何工作的**

首先，下面是*假设*你的密码会发生什么。当 [宣布错误](https://blog.twitter.com/official/en_us/topics/company/2018/keeping-your-account-secure.html) 时，Twitter 首席技术官帕拉格·阿格拉瓦尔解释说，这项服务通常不会存储你的实际密码。当你创建一个 Twitter 账户或更改你的密码时，Twitter 会通过一种算法对其进行加密，以获得一个“哈希”——一个长字符串，就像一种单向工作的编码翻译。(它可能还“加盐”散列，因此如果两个人使用相同的密码，两个存储的散列也不会相同。)

Twitter 存储的是加密的哈希，而不是你的实际密码。每当你登录时，Twitter 会再次将你输入的密码转换成一个散列，并根据它存储的散列进行检查。如果匹配，它会让你进去。如果它们不匹配，它就不匹配。

虽然您总是可以将密码转换成哈希，但是您无法将哈希转换回密码。(这有点像把奶昔变回草莓和牛奶。)这意味着，如果有人曾经侵入 Twitter 的哈希数据库，他们仍然不会拥有每个人的密码。

因为人们一直在侵入数据库，所以服务不保存用户的实际密码是至关重要的。所以，波顿说，这已经成为一种标准做法，每个计算机科学专业的学生都在学习它。即使是小型的非正式服务通常也会将密码转换成散列。情况并非总是如此。在曝光数百万账户的多起高调的 泄露事件之后，这种情况变得更加普遍。

### 什么会出错

但 Twitter 表示，它曾一度未能做到这一点。博尔顿解释了这是如何发生的:开发人员经常在调试模式下运行他们的软件，这产生了软件所做的一切的详细日志。他说:“当你构建一个应用程序时，你通常需要非常详细的日志来查看正在发生的事情，以便轻松地排除故障和/或验证应用程序是否按预期运行。”

但是有时开发人员会忘记在启用系统之前关闭调试日志。这意味着系统会一直记录它不需要的数据，或者不应该记录的数据。这可能包括未加密的密码。博尔顿说，这可能是推特上发生的事情。(我们请推特证实；他们拒绝置评。)

### **推特的回应**

根据博尔顿的说法，虽然这个错误“非常不专业”，但 Twitter 的反应并不专业:尽管风险很低，但他们还是提醒了用户，理论上他们可以隐藏事件。虽然 Twitter 可能不会因为做了正确的事情而成为英雄，但他们肯定比 Equifax 好得多，Equifax 试图隐瞒其数据泄露数月，而其高管 [在可能的内幕交易案件中悄悄出售了他们的股票](https://www.nytimes.com/2018/03/14/business/equifax-executive-insider-trading.html) 。美国证券交易委员会最近对雅虎罚款 3500 万美元 ，原因是其隐瞒数据泄露，暴露了数十亿账户。

博尔顿说，有时候，对数据泄露或错误保持沉默实际上是合适的。如果苹果发现了一个漏洞，并需要一周时间来修复它，那么在修复可用之前保密可能更安全。否则黑客将有一周的自由时间来利用这个漏洞。(这些选择的伦理是安全界高度争论的。)但在像 Twitter 这样的情况下，解决方案是立即可用的，最好是通知公众。

博尔顿说，密码出现在 Twitter 现已删除的内部日志之外的风险很小。(否则 Twitter 需要*强制*每个人修改密码，而不仅仅是建议。)但是总有一点小风险。如果你今天不锁前门，你很可能*没事，但是为什么要冒这个险呢？所以请更改您的密码，如果您在其他地方使用过，也请更改密码。(并且永远不要再重复使用密码。)让你的密码长一点，把它存在密码管理器里。并启用双因素身份验证，这样黑客登录您的帐户时需要的不仅仅是密码。*
