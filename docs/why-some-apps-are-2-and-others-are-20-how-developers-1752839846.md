# 为什么有些应用程序是 2 美元，有些是 20 美元:开发者如何给软件定价

> 原文：<https://lifehacker.com/why-some-apps-are-2-and-others-are-20-how-developers-1752839846>

决定让一个应用程序免费，另一个五美元，还是让另一个按月订阅，需要考虑很多因素。我们大多数人都看不到这一点，但开发商在选择价格标签时会面临艰难的选择——这些选择可能会让他们变得富有，只是支付他们的账单，或者让他们一文不名。

好的软件需要大量的工作。当人们抱怨“贪婪的开发者”竟敢对他们自己开发的应用收取 0.99 美元时，毫无疑问，一些最好的软件是要花钱的，但是大量优秀的软件是免费下载的。为了了解应用程序定价疯狂的方法，我采访了 [布莱恩·穆勒](http://lifehacker.com/im-brian-mueller-and-this-is-the-story-behind-carrot-1454800997) ， [胡萝卜系列应用程序](http://www.meetcarrot.com/) ， [迈克尔·西蒙斯](http://lifehacker.com/behind-the-app-the-story-of-fantastical-1569811300) ，at [Flexibits](https://flexibits.com/) (以 [奇幻](https://flexibits.com/fantastical) 闻名)，以及来自 [扎布卡特](http://zabkat.com/) 的尼科斯

### 市场趋势定义了“合理的价格”

我采访过的每个人在选择价格时都会问自己一系列类似的问题:

*   *目前有哪些与我的应用程序相似的应用程序在销售？*
*   与我竞争的其他应用相比，我的应用有何不同？
*   此类应用最常见的价格是多少？
*   我会为此付出什么代价？我能给客户带来什么价值？

把你在那里收集的东西拿出来，你就有了一个很好的应用程序价格的起点。

“Nikos”描述了这个过程:

> 像我这样的大多数软件“独角戏”对营销一无所知。因此，你检查竞争，看看什么是“合理的”你卖的软件。我开始卖得低一些，但不会低太多，因为许多人把便宜等同于垃圾...

这是一个简单的概念:我们把质量和价格联系在一起。如果你看到一些免费甚至 0.99 美元的东西，很难想象会过分依赖它，因为它看起来只是另一个廉价的一次性应用程序。对于做一件特定事情的应用程序，或者一些小众、愚蠢的软件来说，这可能没什么问题，但是当你在寻找一些重要的东西，比如日历、待办事项应用程序或浏览器替代品时，人们会将抛光和质量与价格标签联系起来。

例如，尽管胡萝卜采取了一种非常愚蠢的方法来提高效率，但它仍然每次都能为我完成任务，这正是我对 3 美元范围内的应用程序的期望。

### 市场份额和我们对功能的期望使得小众应用也变得昂贵

定价也关乎市场份额。Flexibits 和 Mueller 都有适用于 Mac 和 iOS 的应用程序。在这两种情况下，Mac 应用程序的购买成本更高，制作成本也更高。这至少部分是因为 OS X 的市场份额远小于 iOS。换句话说，世界上有成千上万的 iOS 设备，却没有苹果电脑那么多。这意味着开发者需要将 OS X 版本的价格定得更高，以获得可观的开发回报，而 iOS 应用的价格可以更低，以弥补数量上的不足。

桌面应用和移动应用还有一个很大的区别:用户对功能的期望。iPhone 上的 fantastic、iPad 上的 fantastic 和 Mac 上的 fantastic 可能看起来都很相似，但它们的操作方式完全不同。特别是 Mac 应用程序完全不同于移动应用程序，它的操作更加符合我们对“全功能”Mac 日历的期望。我们倾向于从我们的桌面应用程序中期待更多，为了实现这一点，开发者需要要求更高的价格作为回报。

同样，没有什么比缺乏通用应用程序，或者一次购买就能在所有设备上获得相同应用程序更让应用程序购买者恼火的了。Simmons 为 iPhone、iPad 和 OS X 提供了不同的非通用版本的 Fantastical，他解释了这背后的逻辑，以及为什么它们的价格不同

> 我们把应用分开的最大原因是因为我真的相信当你设计一个通用的二进制应用时，你不是在创造一个纯粹的 iPad 应用或者一个纯粹的 iPhone 应用。换句话说，应用程序设计的使命发生了变化。我们专门为 iPhone 设计了 Fantastical。这是一款专为移动设备打造的产品，方便您在旅途中观看。Fantastical for iPad 是专门设计的便携式应用程序，你会花更多的时间，甚至可能坐在你的桌面上。对于 Mac 版本，它是一个桌面应用程序，我们也是这样设计的。所有这些不同的版本允许我们设计更多功能和更好体验的应用程序。
> 
> 另外，iPad 市场比 iPhone 市场小得多，所以销量会更低。桌面应用程序也是如此。

这里归结为基础数学。产品越小众，单位销量就越少，价格就越高。除此之外，我们倾向于期望桌面应用程序有更多的功能(可以说 iPad 应用程序也是如此)，这些功能不会神奇地出现，它们必须被构建。这意味着桌面应用程序处于尴尬的境地，市场份额较小，但对功能的期望最高。综合来看，这意味着桌面软件的价格趋势高于类似的——甚至相同的——移动应用。

### 应用程序的价格必须涵盖 API 和其他底层服务

当一个应用程序链接到第三方服务时，通常需要花钱，而这些钱必须来自某个地方。比如说，大多数开发者都使用第三方服务，而不是构建自己的实时天气或地图服务，这样他们就不必自己重新发明轮子了。当然，这增加了应用程序的成本。布莱恩·穆勒遇到了这种与 [胡萝卜天气](http://lifehacker.com/carrot-weather-adds-a-little-humor-to-your-weather-fore-1692378982) :

> *【天气】有与其天气数据 API 相关的成本。所以我不得不做一点数学计算，算出每个用户使用这款应用的频率，并把它算进基本价格。但它最终还是和其他高质量的天气应用一样。*

比如胡萝卜天气用的 [Forecast.io](http://forecast.io/#/f/34.0535,-118.2453) ，这也恰好是 iOS 上我们最喜欢的天气 app[、暗黑天空](https://lifehacker.com/the-best-weather-app-for-iphone-5982744) 的主干。下面是使用 Forecast.io 的 [定价模型:](https://developer.forecast.io/)

> 您可以在商业和非商业应用程序中使用 API(应用程序编程接口)。
> 
> 1.  你每天进行的前 1000 次 API 调用都是免费的。
> 2.  之后的每个 API 调用都要花费 0.0001 美元。
> 3.  *给我们一个“Powered by Forecast”徽章，链接到*[*【http://forecast.io/】*](http://forecast.io/)*无论您在哪里显示来自 API 的数据。*

“每个 API 调用 0.0001 美元”听起来可能不多，但如果你的应用程序变得流行(如胡萝卜天气)，它会很快增加。只要有人的 iPhone 上有胡萝卜天气，就拨入 Forecast.io，这就被算作 API 调用。如果你有 1000 部手机每天签到一次，这是一回事，但是如果你有 1000 部手机一整天都在签到，一天多次，以保持天气面板或小部件更新，那么，这是大量的 API 调用，和一个沉重的账单。一个 app 卖的越多，API 调用出去的就越多，维护的成本就越高。

这个天气 API 只是一个例子，但是还有很多其他的例子。 [谷歌地图](https://developers.google.com/maps/faq?hl=en#usage_pricing) 、 [Twilio](https://www.twilio.com/pricing) 和 [谷歌日历](https://developers.google.com/google-apps/calendar/pricing) 是另外几个基于使用情况计费的流行服务，大多数人不会下载不连接这些流行服务的地图应用、日历应用或消息应用。

### 长期支持和更新不是免费的

每个制作应用程序的人都希望它成功，但更多的用户意味着更多的责任来支持他们。随着你的应用程序越来越受欢迎，用户想要新的功能，遇到错误，或者只是挣扎着使用它。大多数开发人员最终处理他们自己的支持请求，这变成了一个相当大的时间浪费。只要看看 [【胡萝卜】](https://twitter.com/CARROT_app/with_replies) 或[flexi bits](https://twitter.com/flexibits/with_replies)Twitter 账户就能窥见其中一个支持渠道。所有的时间都是金钱。

这是我从来没有想到的定价最大的方面。对于自由软件，我们的期望很低。我知道当一个新的操作系统出来时，它不会立即得到更新。除了发布之外，我对支持没有任何期望，并期待应用程序有怪癖。对于付费软件，我想要更好的。我预计它在推出后会很快与新的操作系统兼容。我预计虫子会在几周内被压扁，而不是几个月。我希望，如果我通过社交媒体或电子邮件联系开发商，我会尽快收到回复。我也期待新的功能经常出现，奇怪的怪癖得到解决。西蒙斯阐述道:

> *【我们扪心自问】我们在向客户回报什么样的价值？最终，我们为客户开发应用程序...我们绝对在乎我们卖的是什么。我们有一个支持团队，但肯特和我打开我们的支持，每天都看，这样我们就知道人们在抱怨什么。支持肯定是一种成本，但这是我们关心的事情，因为当我们看到它时，我们可以使我们的应用程序变得更好。*

“支持”的意思是多方面的。这意味着通过电子邮件或社交媒体提供技术支持。这意味着添加客户要求的新功能。这意味着更新一个应用程序以在新版 iOS 或 Android 上工作，并利用这些版本中可用的新功能。简单来说，这是一个复杂的过程，在你点击购买按钮后并没有结束，而是耗费了开发者的时间和金钱。

我接触过的每一个开发者都说，归根结底，他们想赚足够的钱，这样他们就可以开发另一个应用。之后又是一个。这与把*变成*一个应用程序所需的时间无关(几乎每个人都说，如果按时间计算，他们的应用程序会贵得惊人)。只是做他们喜欢做的事情。
