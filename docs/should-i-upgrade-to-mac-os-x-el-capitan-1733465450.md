# 我应该升级到 Mac OS X El Capitan 吗？

> 原文：<https://lifehacker.com/should-i-upgrade-to-mac-os-x-el-capitan-1733465450>

亲爱的生活黑客，
[OS X El Capitan](https://www.apple.com/osx/elcapitan-preview/) 今天出了。看起来不错，但还是升级，任何升级都有可能变坏。你们一直在用它。你怎么想呢?是准备好黄金时间了，还是我应该等待下一个补丁来修复人们不可避免会发现的 bug？

亲爱的船长，在这一点上有一件事是清楚的:苹果对 OS X 的外观、感觉和功能非常满意。他们在过去几个更新中所做的改进是严格进化的。他们所做的一切都没有太大的改变，尽管 El Capitan 肯定拥有一些有用的新功能，但总体而言，你不会看到过去两三个 OS X 版本之间的巨大差异。

更重要的是:如果你记得 OS X 雪豹与 OS X 雪豹相比在性能和可用性方面的全面提升，你会立刻理解升级到埃尔卡皮坦与优胜美地相比是什么感觉。

事实上，甚至不是约塞米蒂有问题需要解决。优胜美地相当坚固。然而，El Capitan 主要是在引擎盖下进行性能调整，对捆绑的苹果应用程序进行功能更新，以及一些有用的额外功能，这些功能可能会也可能不会成为您常规工作流程的一部分。如果你正在寻找令人惊讶的新功能，这里没有——但是你可能会找到一些修复到目前为止你刚刚插入的持久和恼人的错误的方法。

### 谁应该立即升级

让我们马上开始:除非你有理由*不*更新，这是一个值得的升级。大多数改进都是在速度、可用性和更少崩溃方面，这对每个人都很好。El Capitan 的 [系统要求与 Yosemite(和 Mavericks，就此而言)相同，所以如果你正在阅读这篇文章，你有一台 Mac 电脑可以处理它。此外，El Capitan 广泛的公共测试期给了开发人员充足的时间来确保他们的应用程序和服务兼容并准备就绪。](http://osxdaily.com/2015/06/09/os-x-el-capitan-system-requirements-compatible-mac/)

我们已经在 Lifehacker 进行了几个月的公开测试，除了这里或那里的一些小问题，体验还是不错的。即使没有明确说明 El Capitan 兼容性的应用程序，也和它们在 Yosemite 中的工作方式一样。一旦其中一些应用程序获得了 El Capitan 兼容性，它实际上只是支持 El Capitan 的新分屏功能。事实上，除了更可靠之外，它并没有给人一种来自约塞米蒂的巨大改变。

毫无疑问，El Capitan 带来的新功能绝对有用。分屏非常适合窗口工作，OS X 早就应该这样做了。如果 OneNote 和 Evernote 有一个轻量级的 iCloud 同步婴儿，新的 Notes 应用程序很像你会得到的东西。如果你有一台运行 iOS 9 的 iPhone 或 iPad，那就非常适合随身携带这些笔记。埃尔卡皮坦的虚拟桌面和任务控制比在约塞米蒂要快得多，如果你有一个鼠标 [可以轻松使用它们](http://lifehacker.com/the-logitech-mx-master-a-fitting-successor-to-one-of-o-1713362467) ，它们会比以前更有用。Spotlight 的新功能无疑也让它在 El Capitan 中更加有用(尽管仍然无法取代我们非常喜爱的[阿尔弗雷德](http://www.alfredapp.com/) )。)

如果你依赖这些功能，或者它们听起来对你有吸引力，你会很高兴开箱升级。即使它们对您来说并不特别，您也会欣赏速度和性能的提高。El Capitan 在我 2010 年中期的 Macbook Pro 上运行得比 Yosemite 更快更流畅，肯定比 Mavericks 好，这说明了它有多旧。

### 谁应该推迟一段时间

当然，除非你*有*在流血边缘，等几天后发射。让您喜爱的应用程序背后的开发人员进行更新，回复错误报告，并将更新的版本推送到 Mac App Store。没有那么严重，我们建议你等待苹果的更新，但如果你偶然发现某个应用程序真的真的坏了，开发者坚持认为是苹果的错，你可能会想推迟。检查一下 [Roaring Apps 的海量兼容性图表](https://roaringapps.com/apps) 看看你每天使用的程序是否都准备好了，以防万一。请记住，他们的列表并不总是最新的，所以如果你没有看到任何东西，请查看开发人员的博客或 Twitter 或脸书，看看他们在说什么。很有可能他们正在谈论它，如果苹果还没有批准他们应用的更新，他们肯定正在谈论它。

在 El Capitan 中你可能需要注意的一件事是一个名为 [系统完整性保护](https://en.wikipedia.org/wiki/System_Integrity_Protection) (SIP)的新安全功能，它有可能破坏一些应用程序。简而言之，苹果引入它是为了限制根用户和进程的访问级别，并保护操作系统的核心组件。这意味着有些事情你不能做，即使你有并使用 root。你可以在 Ars Technica 阅读 [的细节。这也意味着一些利用 root 的应用程序将不再工作——一个显著的例子是酒保，在它的](http://arstechnica.com/apple/2015/09/os-x-10-11-el-capitan-the-ars-technica-review/8/#h1) [最近更新了 El Capitan 支持](http://lifehacker.com/bartender-2-for-mac-adds-keyboard-shortcuts-search-an-1733370847) 。这不是一个大问题，你可以禁用它，但实际上大多数人不需要也不应该这么做。然而，一些程序可能需要更新才能工作——如果你正在使用一些已经工作了很长时间的废弃应用程序，而它们崩溃了，那么它们可能会永远死去，你必须打电话禁用它或找到替代程序。

除此之外，与 Yosemite 不同的是，一些著名的第三方应用程序在发布时间 出现问题，我们在 El Capitan 没有看到任何类似的问题。我在 Yosemite 报告的图形和 QuickLook 迟缓问题已经得到了修补，虽然谷歌搜索确实在 Yosemite 和 El Capitan 发现了一些 GPU 问题(特别是在旧系统中)，但我个人无法报告任何问题。我遇到了 [一个旧的 VirtualBox 安装](https://forums.developer.apple.com/thread/13245) 的问题，它导致了一些可怕的内核恐慌，但没有任何其他严重的问题。即便如此，就像我们常说的那样:如果你每天都用你的 Mac 电脑工作，并且不能忍受由于拙劣的升级而导致的停机，确保你的数据都备份好了，如果你遇到意想不到的问题，在你可以负担得起让你的 Mac 电脑离线一会儿的时候进行升级(或者更好的是，全新安装)。

### 谁可能根本不想升级

到目前为止，我们已经注意到埃尔卡皮坦几乎适合所有人。每一个拥有现代 Mac 的人，也就是想要升级的人。当然，有些人根本不应该升级，但我们愿意打赌你已经知道自己是谁了。

如果你有一些必备的应用程序不能在 OS X 的新版本中运行(或者不能运行，你已经跳过了更新)，它们可能不能在 El Capitan 中运行，所以你最好留在原地。如果你运行的是 Mavericks 之前的 Mac，你对它和它的性能很满意，或者你处于系统需求的边缘，想知道 El Capitan 在你的机器上是否会比 Mavericks 或 Yosemite 更容易，它可能不会。如果你真的如此渴望，你可以使用磁盘工具或 [副本复制程序](http://bombich.com/) 来制作你当前系统的映像，保存到备份硬盘，然后安装 El Capitan 来试用。如果它不起作用，就用你的备份重新映像你的 Mac。尽管如此，这仍然是一个很大的麻烦，如果你不愿意为约塞米蒂做这件事，在埃尔卡皮坦没有什么值得做的。我们建议你继续使用那些能让你高效工作、让你的 Mac 可用和开心的东西。

### 底线是:把它当作一个服务包，只是先备份你的数据

像所有操作系统一样，我们最好的建议很简单:在发布后等几天再升级。花时间确保你的数据都备份了(并且你已经测试了恢复，所以你知道你的备份实际上是可行的)，并确保你最重要的应用程序是兼容的，或者已经更新以利用 El Capitan 的时髦新功能。然后，留出一些时间，你可以不用你的 Mac 来做升级。这样，如果一切顺利，你马上就可以回去工作了。如果一切顺利，你将有时间解决问题。

在许多方面，El Capitan 感觉非常像一个服务包或增量升级，很难像一个全新的操作系统那样谈论它。很难告诉你做一个全新的安装，或者在升级之前确保你已经备份好，扣好安全带，并且你的托盘桌处于直立位置。即便如此，你比我们更了解你的系统。如果你喜欢反复无常，你可以从应用商店升级。如果你像我们一样，喜欢你的干净安装，做你的尽职调查。无论哪种方式，一旦你完成了，你会有一个更快，更快的 Mac。
